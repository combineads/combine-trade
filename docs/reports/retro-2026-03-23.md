## 어려웠던 것 / 개선 여지

**1. QA 주기 및 리뷰 데이터 없음** — `.harness/state/qa-cycles-*.json`, `review-rounds-*.json` 미생성. 태스크별 QA 반복 횟수, 리뷰 첫 통과율을 측정할 수 없음. 하네스 학습 루프의 가장 큰 데이터 공백.

**2. 초기 에픽들의 배치 커밋** — EP00~EP14는 에픽 단위 단일 커밋으로 묶임 (`feat: implement EP05 backtest engine`). 에픽 내 태스크별 변경사항을 git blame으로 추적하기 어려움. T-128 이후부터 태스크 단위 커밋으로 전환되어 개선됨.

**3. 범위 확장 3회** — EP16(매크로/LLM), EP17(Double-BB), EP18(better-auth 멀티유저)가 부트스트랩 계획에 없었음. 초기 계획 대비 약 55% 태스크 초과 (83 → 183). 각각 정당한 추가였지만, 초기 계획 시점에 식별됐더라면 일정 예측이 더 정확했을 것.

**4. apps/desktop 미구현** — 아키텍처 문서에 Tauri 데스크탑 앱 통합이 상세히 설계돼 있으나 실제 `apps/desktop/` 디렉터리가 존재하지 않음. `packages/ui/src/`의 platform/ 구조도 원래 설계와 다르게 구현됨 (auth/, stores/, theme/, lib/로 대체).

**5. userId 추출 패턴 중복** — `apps/api/src/routes/*.ts` 전반에 동일 패턴 12회 이상 반복. 리뷰에서 잡혔어야 할 코드 냄새.

## 권고사항

**1. QA·리뷰 상태 파일 생성 활성화** (최우선)

**2. apps/desktop Tauri 스캐폴드** (중요도: 중)
아키텍처 설계는 완료됐으나 실제 디렉터리가 없음. `packages/ui/src/`의 플랫폼 어댑터 코드가 실제 Tauri 환경에서 검증되지 않았음. EP19로 계획 권장.

**3. userId 미들웨어 추출** (중요도: 중)
`apps/api/src/routes/*.ts` 전반의 userId 추출 패턴 12회 중복. Elysia `derive()` 플러그인으로 한 번에 처리하는 것이 안전하고 간결함. 개별 태스크로 처리 권장.

**4. p99 레이턴시 CI 테스트 추가** (중요도: 중)
1초 예산 달성 여부가 CI에서 자동으로 측정되지 않음. 의사결정 파이프라인 end-to-end 벤치마크를 CI 게이트에 추가하면 레이턴시 회귀를 조기 감지 가능.

**5. done/ 태스크 아카이브** (낮음)
183개 태스크가 `docs/tasks/done/`에 누적됨. 에픽 단위로 `docs/tasks/archive/<epic-slug>/`에 이동하고 SUMMARY.md 생성 권장. 컨텍스트 크기 관리에 효과적.
