## Patterns discovered

### [New] 에픽 리뷰 시 코드 실사 필수
- **Observed in**: EP-15 리뷰 (Round 5)
- **Category**: efficiency
- **Description**: 에픽 초안 리뷰에서 코드를 직접 읽고 실행 흐름을 추적한 결과, 문서만으로는 발견 불가능한 3중 단절(A-grade), 함수명 오류(calculateTransfer vs calculateTransferable), 존재하지 않는 키(wick_ratio) 등 13건을 발견
- **Root cause**: 에픽 계획은 코드 "상태"를 기술하나, 실제 코드 "동작"과 차이 발생
- **Recommendation**: 에픽 계획 후 반드시 코드 실사 기반 리뷰를 수행. 특히 모듈 간 데이터 흐름(A→B→C)에서 실제로 값이 전달되는지 grep + read로 확인

### [New] 시그니처 변경은 호출부 전수 검색 필수
- **Observed in**: T-15-008 (makeDecision 시그니처)
- **Category**: efficiency
- **Description**: 공개 함수 시그니처 변경 시 7개 테스트 파일 + 2개 소스 파일의 모든 호출부를 갱신해야 함. grep 전수 검색 없이는 누락 발생
- **Root cause**: TypeScript의 타입 시스템이 컴파일 시 잡지만, 테스트의 mock/stub은 런타임에서만 확인 가능
- **Recommendation**: 시그니처 변경 태스크는 항상 `grep -r "함수명" src/ tests/` → 전수 갱신을 Steps에 포함

### [Recurring] normalizer 분포 무관성
- **Observed in**: T-15-005 (EP-15), EP-10 (strategy alignment)
- **Description**: Median/IQR 정규화는 입력 분포에 무관하므로 벡터 구조 변경에도 수정 불필요
- **Recommendation**: 향후 벡터 구조 변경 시 normalizer는 "검증만" 태스크로 계획 (구현 태스크 불필요)
