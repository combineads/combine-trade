# T-02-005 indicators/atr.ts — ATR(14) calculation

## Goal
Implement Average True Range calculation wrapping `@ixjb94/indicators` IndicatorsSync.atr(). Default period is 14 (industry standard).

## Why
ATR measures volatility and is used for stop-loss sizing, position sizing risk calculations, and as a vectorization feature. It's critical for determining appropriate SL distances relative to current volatility.

## Inputs
- `src/indicators/types.ts` (T-02-001) — ATRResult type (Decimal | null)
- `src/core/decimal.ts` — d() factory
- `@ixjb94/indicators` — IndicatorsSync.atr(high, low, close, period)

## Dependencies
T-02-001 (indicators/types.ts)

## Expected Outputs
- `src/indicators/atr.ts` — calcATR() function
- Used by position sizing, SL distance calculation, vectorization

## Deliverables
- `src/indicators/atr.ts`

## Constraints
- L2 module: imports from `core/` and `indicators/types`
- ATR is always positive (absolute volatility measure)
- Default period = 14 (industry standard)
- Requires high[], low[], close[] arrays (3 separate inputs)
- Return null when insufficient data

## Steps
1. Import IndicatorsSync from `@ixjb94/indicators`
2. Import d, Decimal from `@/core/decimal`
3. Define ATR_DEFAULT_PERIOD = 14
4. Create `calcATR(highs: number[], lows: number[], closes: number[], period: number = ATR_DEFAULT_PERIOD): Decimal | null`:
   - Return null if arrays.length < period + 1
   - Validate all 3 arrays have same length
   - Call IndicatorsSync.atr(highs, lows, closes, period)
   - Take last non-NaN value
   - Return as Decimal (should always be positive)
5. Create `calcATRSeries(highs: number[], lows: number[], closes: number[], period: number = ATR_DEFAULT_PERIOD): Decimal[]`:
   - Full series, filter NaN
6. Export calcATR, calcATRSeries, ATR_DEFAULT_PERIOD
7. Write tests
8. Verify typecheck passes

## Acceptance Criteria
- calcATR returns positive Decimal value
- Returns null when insufficient data
- Higher volatility candles → larger ATR
- Constant H/L/C → ATR approaches 0
- `bun run typecheck` passes

## Test Scenarios
- calcATR with 15+ candles having known H/L/C → returns positive Decimal
- calcATR with 14 candles (period=14) → returns null (needs period+1)
- calcATR with high-volatility data (large H-L spread) → returns larger value
- calcATR with constant H=L=C → returns d("0") or very small value
- calcATR result is always non-negative
- calcATRSeries returns array of Decimal values with correct length
- calcATR with mismatched array lengths → throws or handles gracefully

## Validation
```bash
bun run typecheck
bun test --grep "atr"
```

## Out of Scope
- True Range calculation (internal to @ixjb94/indicators)
- ATR-based trailing stop logic (EP-07)
