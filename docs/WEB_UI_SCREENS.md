# Web UI 화면 설계

## 개요

combine-trade Web UI는 1인 운영자가 "지금 잘 돌고 있나?"를 확인하고 긴급 조치를 취하는 데 집중합니다.
분석/디버깅은 터미널과 DB 쿼리로, 알림은 Slack으로 처리하므로 웹은 최소 화면만 유지합니다.

## 화면 목록

| # | 화면 | 목적 |
|---|------|------|
| 1 | 로그인 | 단일 사용자 비밀번호 인증 |
| 2 | 대시보드 | 시스템 상태 + 활성 포지션 + 오늘 PnL + 모드 전환 + 킬스위치 |
| 3 | 거래 내역 | 완료 거래 테이블 + 승률/기대값/MDD 요약 |

## 1. 로그인

### Google Stitch 프롬프트

```
다크 테마 트레이딩 시스템 로그인 페이지를 만들어줘.

[디자인 스타일]
- 배경색: #0a0e14 (어두운 네이비 블랙)
- 카드 배경: #1e293b
- 테두리: #334155
- 기본 텍스트: #f1f5f9
- 보조 텍스트: #94a3b8
- 프라이머리 색상(버튼): #17b862 (에메랄드 그린)
- 버튼 호버: #0a954e
- 폰트: Inter (본문), JetBrains Mono (숫자/코드)
- 다크 모드 전용, 라이트 모드 없음

[레이아웃]
- 화면 정중앙에 로그인 카드 1개 (최대 너비 400px)
- 카드 위에 로고 영역: "COMBINE TRADE" 텍스트 로고 (font-weight: 700, 크기: 24px, 색상: #17b862)
- 로고 아래 부제: "Double-BB 자동매매 시스템" (크기: 14px, 색상: #94a3b8)

[카드 내용]
- 비밀번호 입력 필드 1개 (라벨: "비밀번호", placeholder: "비밀번호를 입력하세요…")
  - 입력 필드 배경: #0f172a, 테두리: #334155, 텍스트: #f1f5f9
  - 포커스 시: 테두리 #17b862 + outline 2px solid #17b862, outline-offset 2px
- "로그인" 버튼 (전체 너비, 배경: #17b862, 텍스트: 흰색, font-weight: 600)
  - 호버: #0a954e
  - 누름(pressed): #097840
  - 포커스: outline 2px solid #17b862, outline-offset 2px
- 카드 하단 텍스트: "v0.1.0" (크기: 12px, 색상: #64748b)

[에러 상태]
- 비밀번호 틀렸을 때: 입력 필드 테두리 #ef4444, 필드 바로 아래 에러 메시지 "비밀번호가 일치하지 않습니다. 다시 입력해 주세요." (색상: #ef4444, 크기: 13px)
- 에러 발생 시 입력 필드에 자동 포커스

[로딩 상태]
- 로그인 버튼 클릭 후: 버튼 텍스트 "로그인 중…" + 스피너, 버튼 비활성화 (opacity 0.5)

[참고]
- 사용자 1명만 사용하므로 아이디 필드 없음, 비밀번호만
- Enter 키로 제출 가능 (<form> 래핑)
- 불필요한 장식 없이 깔끔하게
```

- 포커스 트랩: Tab 키가 모달 내부에서만 순환 ("취소" ↔ "긴급 청산 실행")

3. "오늘 손실 한도" — 프로그레스 바: "2.1% / 10%" (여유 있으면 초록, 70% 이상 노랑, 90% 이상 빨강)

## 구현 시 체크리스트 (Web Interface Guidelines)

Stitch 디자인 완료 후, 코드로 구현할 때 반영할 항목:

### HTML 기본

- [ ] `<html lang="ko">` 한국어 설정
- [ ] `<html>` 에 `color-scheme: dark` 설정
- [ ] `<meta name="theme-color" content="#0a0e14">`
- [ ] 페이지 전환 시 `<title>` 업데이트 ("대시보드 — COMBINE TRADE", "거래 내역 — COMBINE TRADE")

### 접근성

- [ ] 아이콘 전용 버튼에 `aria-label` 추가
- [ ] 장식용 아이콘에 `aria-hidden="true"`
- [ ] 폼 컨트롤에 `<label>` 또는 `aria-label` 연결
- [ ] 테이블에 시맨틱 마크업 (`<table>`, `<thead>`, `<th scope="col">`)
- [ ] 헤딩 계층 유지 (`<h1>` → `<h2>` → `<h3>`)
- [ ] 스킵 링크 (`<a href="#main">본문으로 건너뛰기</a>`)
- [ ] 토스트/알림에 `aria-live="polite"`
- [ ] Trade Block 토글에 `role="switch"` + `aria-checked`
- [ ] 네비게이션: `<nav aria-label="메인 네비게이션">`
- [ ] 페이지네이션: `<nav aria-label="페이지네이션">`
- [ ] 기간 탭: `role="tablist"` + `role="tab"` + `aria-selected`
- [ ] 상태를 색상만이 아닌 텍스트로도 전달 (색맹 대응)

### 포커스

- [ ] 모든 인터랙티브 요소에 `:focus-visible` 스타일 (`:focus` 대신)
- [ ] `outline: none` 단독 사용 금지
- [ ] 킬스위치 확인 모달에 포커스 트랩
- [ ] 모달 열릴 때 첫 번째 포커스 가능 요소로 포커스 이동
- [ ] 복합 컨트롤에 `:focus-within` 활용

### 폼

- [ ] 비밀번호 필드: `type="password"`, `autocomplete="current-password"`, `spellCheck={false}`
- [ ] 제출 버튼: 요청 중 스피너 + 비활성화, 중복 제출 방지
- [ ] 에러 메시지: 필드 인라인, 첫 에러에 포커스
- [ ] `<select>`: 명시적 `background-color` + `color` (Windows 다크모드 대응)

### 성능

- [ ] Google Fonts: `<link rel="preconnect" href="https://fonts.googleapis.com">` + `font-display: swap`
- [ ] 중요 폰트: `<link rel="preload" as="font">`
- [ ] 이미지 사용 시 `width`/`height` 명시
- [ ] CDN 도메인: `<link rel="preconnect">`

### 네비게이션

- [ ] 필터/탭/페이지네이션 상태를 URL 쿼리 파라미터에 반영
- [ ] 모든 링크는 `<a>`/`<Link>` (Cmd+클릭, 가운데 클릭 지원)
- [ ] 킬스위치: 확인 모달 필수 (즉시 실행 금지)
- [ ] 실거래 모드 전환: 확인 모달 필수

### 애니메이션

- [ ] WATCHING pulse 애니메이션: `prefers-reduced-motion` 대응 (비활성화)
- [ ] `transition` 속성 명시 (`transition: all` 금지)
- [ ] 가격/PnL 숫자 변경은 애니메이션 없이 즉시 업데이트
- [ ] `transform-origin` 올바르게 설정
- [ ] 애니메이션은 `transform`/`opacity`만 사용 (compositor 친화)

### 타이포그래피

- [ ] 말줄임표: `…` 사용 (`...` 금지)
- [ ] 숫자 열: `font-variant-numeric: tabular-nums`
- [ ] 긴 텍스트: `truncate` 또는 `line-clamp` 처리
- [ ] flex 자식 요소: `min-w-0` (텍스트 truncation 허용)

### 다국어/포맷

- [ ] 날짜: `Intl.DateTimeFormat` 사용
- [ ] 숫자/통화: `Intl.NumberFormat` 사용

### 터치

- [ ] `touch-action: manipulation` (모바일 더블탭 방지)
- [ ] 모달에 `overscroll-behavior: contain`
- [ ] `-webkit-tap-highlight-color` 의도적 설정
