# T-13-013 백테스트 CLI 인터페이스

## Goal
`bun run backtest` CLI 진입점을 구현한다. 옵션 파싱, 진행률 표시, 결과 요약을 포함.

## Why
백테스트/WFO를 터미널에서 실행하는 사용자 인터페이스. PRODUCT.md의 "Backtesting (same code path as live)" 기능 접근점.

## Inputs
- `src/backtest/engine.ts` — BacktestRunner
- `src/backtest/wfo.ts` — runWfo
- `src/backtest/reporter.ts` — printReport
- `package.json` — scripts.backtest 설정

## Dependencies
- T-13-005, T-13-008, T-13-011

## Expected Outputs
- `src/backtest/cli.ts` — CLI 진입점
- `package.json` scripts.backtest 업데이트

## Deliverables
- `src/backtest/cli.ts`
- `package.json` 업데이트

## Constraints
- 옵션: --symbol (필수), --exchange (기본: binance), --start (필수), --end (필수), --mode (backtest|wfo, 기본: backtest), --threads (기본: CPU/2)
- 진행률: 현재 캔들/총 캔들, WFO 시 현재 윈도우/총 윈도우, ETA
- 에러: 잘못된 심볼, 날짜 범위 오류 등 명확한 메시지
- `bun run backtest`으로 실행 가능

## Steps
1. Write test code from ## Test Scenarios (RED phase)
2. Run tests — confirm all behavioral tests fail
3. `src/backtest/cli.ts` 작성 — `Bun.argv` 기반 argparse
4. 옵션 파싱 및 검증 (누락 필수 옵션, 날짜 형식 등)
5. --mode=backtest → BacktestRunner 실행 + printReport
6. --mode=wfo → runWfo 실행
7. 진행률 표시: stderr에 업데이트 (캔들 처리 콜백에서)
8. `package.json` scripts.backtest = "bun src/backtest/cli.ts"
9. Run tests — confirm all pass (GREEN phase)
10. Refactor while keeping tests green (REFACTOR phase)

## Acceptance Criteria
- `bun run backtest -- --help` → 옵션 목록 출력
- `bun run backtest -- --symbol BTCUSDT --start 2024-01-01 --end 2024-06-01` → 백테스트 실행 + 결과
- `bun run backtest -- --mode wfo --symbol BTCUSDT --start 2021-01-01 --end 2024-01-01` → WFO 실행
- 잘못된 옵션 → 에러 메시지 + 사용법 출력

## Test Scenarios
- parseArgs(["--symbol", "BTCUSDT", "--start", "2024-01-01", "--end", "2024-06-01"]) → 정상 파싱
- parseArgs([]) → 에러 "symbol is required"
- parseArgs(["--start", "invalid-date"]) → 에러 "invalid date format"
- parseArgs(["--start", "2024-06-01", "--end", "2024-01-01"]) → 에러 "start must be before end"
- parseArgs(["--mode", "invalid"]) → 에러 "mode must be backtest or wfo"
- parseArgs(["--threads", "0"]) → 에러 "threads must be >= 1"
