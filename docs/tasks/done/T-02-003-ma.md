# T-02-003 indicators/ma.ts — SMA and EMA calculation

## Goal
Implement Simple Moving Average (SMA) and Exponential Moving Average (EMA) calculations wrapping `@ixjb94/indicators` IndicatorsSync.sma() and .ema(). Support periods 20, 60, 120 as defined in constants.ts.

## Why
MA20/60/120 are used for TP1 target (1H MA20), direction filtering, and trend strength assessment. EMA variants provide faster-responding alternatives used in vectorization features. Both SMA and EMA are fundamental building blocks referenced throughout the trading pipeline.

## Inputs
- `src/indicators/types.ts` (T-02-001) — MAResult type (Decimal[])
- `src/core/constants.ts` — MA_PERIODS [20, 60, 120]
- `src/core/decimal.ts` — d() factory
- `@ixjb94/indicators` — IndicatorsSync.sma(source, period), .ema(source, period)

## Dependencies
T-02-001 (indicators/types.ts)

## Expected Outputs
- `src/indicators/ma.ts` — calcSMA(), calcEMA() functions
- Used by signals (TP1 price = 1H MA20), direction filter, vectorization

## Deliverables
- `src/indicators/ma.ts`

## Constraints
- L2 module: imports from `core/` and `indicators/types`
- Internal calculation uses Float64 via @ixjb94/indicators
- Output values are Decimal
- Accept number[] source array (reuse candlesToCloses from bollinger.ts if needed, or accept raw numbers)
- Return latest value as Decimal | null, or full series as Decimal[]
- Return null when source.length < period

## Steps
1. Import IndicatorsSync from `@ixjb94/indicators`
2. Import d, Decimal from `@/core/decimal`
3. Create `calcSMA(source: number[], period: number): Decimal | null`:
   - Return null if source.length < period
   - Call IndicatorsSync.sma(source, period)
   - Return last non-NaN value as Decimal
4. Create `calcSMASeries(source: number[], period: number): Decimal[]`:
   - Call IndicatorsSync.sma(source, period)
   - Convert entire array to Decimal[], filtering NaN
5. Create `calcEMA(source: number[], period: number): Decimal | null`:
   - Return null if source.length < period
   - Call IndicatorsSync.ema(source, period)
   - Return last non-NaN value as Decimal
6. Create `calcEMASeries(source: number[], period: number): Decimal[]`:
   - Similar to SMA series
7. Export all functions
8. Write tests with known data
9. Verify typecheck passes

## Acceptance Criteria
- calcSMA with period 20 and 20+ values → returns correct Decimal value
- calcEMA with period 20 and 20+ values → returns correct Decimal value
- Returns null when insufficient data
- SMA of constant values equals that constant
- EMA of constant values equals that constant
- Series functions return full array of computed values
- `bun run typecheck` passes

## Test Scenarios
- calcSMA([1,2,3,4,5], 3) → returns d("4") (average of last 3: 3+4+5=12/3=4)
- calcSMA with source.length < period → returns null
- calcSMA with constant source [10,10,10,10,10], period 3 → returns d("10")
- calcEMA with 20+ values → returns Decimal (non-null)
- calcEMA with source.length < period → returns null
- calcSMASeries returns array of Decimal values matching expected count
- calcEMASeries returns array with correct length
- SMA and EMA of same constant data → both return same value

## Validation
```bash
bun run typecheck
bun test --grep "ma"
```

## Out of Scope
- Weighted MA, VWMA (not needed by Double-BB strategy)
- Period validation against constants.ts (caller's responsibility)
