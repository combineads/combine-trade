# T-18-001 better-auth setup and configuration

## Goal
Install `better-auth` and `@elysiajs/helmet`, create the better-auth instance configuration in `packages/shared/auth/better-auth.ts`, and generate the Drizzle schema for the four better-auth tables (user, session, account, verification) — including a migration that integrates the existing `users` table.

## Why
The current `packages/shared/auth/` has two conflicting JWT implementations (`jwt.ts` uses `sub`/`role`, `token.ts` uses `userId`) and a non-functional logout (no real token revocation). better-auth resolves all of these with a battle-tested session model. This task establishes the library installation and schema foundation that every subsequent auth task depends on.

## Inputs
- `docs/exec-plans/18-better-auth-multiuser.md` § M1 — install targets, instance config shape, schema migration requirements
- `db/schema/users.ts` — existing `users` table columns to map: `id`, `email`, `password_hash`, `name`
- `db/schema/index.ts` — export registry to update
- `packages/shared/auth/` — current directory; better-auth instance goes here

## Dependencies
None — first task in the EP18 chain.

## Expected Outputs
- `better-auth` and `@elysiajs/helmet` added to `package.json` (workspace root or `packages/shared`)
- `packages/shared/auth/better-auth.ts` — exported `auth` instance
- `db/schema/better-auth.ts` — Drizzle schema for `user`, `session`, `account`, `verification` tables

## Deliverables

### `packages/shared/auth/better-auth.ts`
```typescript
import { betterAuth } from "better-auth";
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { db } from "../../db/index.js"; // adjust path to actual db singleton

export const auth = betterAuth({
  database: drizzleAdapter(db, { provider: "pg" }),
  emailAndPassword: { enabled: true },
  advanced: {
    cookiePrefix: "combine-trade",
    generateId: () => crypto.randomUUID(),
  },
});

export type Auth = typeof auth;
```

### `db/schema/better-auth.ts`
Schema for the four better-auth tables using Drizzle pg-core, aligned with better-auth's Drizzle adapter expectations:
- `user` table: `id` (text PK), `name`, `email` (unique), `emailVerified` (boolean), `image`, `createdAt`, `updatedAt`
- `session` table: `id`, `expiresAt`, `token` (unique), `createdAt`, `updatedAt`, `ipAddress`, `userAgent`, `userId` (FK → user.id)
- `account` table: `id`, `accountId`, `providerId`, `userId` (FK), `accessToken`, `refreshToken`, `idToken`, `accessTokenExpiresAt`, `refreshTokenExpiresAt`, `scope`, `password`, `createdAt`, `updatedAt`
- `verification` table: `id`, `identifier`, `value`, `expiresAt`, `createdAt`, `updatedAt`

### Migration script note
The migration must handle alignment between the existing `users` table and better-auth's `user` table:
- Rename `users` → `user` OR create `user` and migrate data
- Map `users.password_hash` → `account.password` for `emailAndPassword` provider
- Preserve all existing UUIDs (existing `users.id` values must be kept as `user.id`)

## Implementation Notes

### Package naming deviation
`@elysiajs/helmet` does not exist on npm. The actual package is `elysiajs-helmet` (no scope prefix). Installed `elysiajs-helmet@1.0.4` instead. Callers in `apps/api` should `import { elysiaHelmet } from "elysiajs-helmet"`.

### Factory pattern instead of singleton
The task spec shows a singleton `export const auth = betterAuth(...)` that hard-imports the DB singleton. To maintain the `packages/shared` → `db/` dependency direction (shared must not import db), the implementation uses a `createAuth(config)` factory. The DB adapter is injected at runtime by the caller (`apps/api`), not imported here.

### `db:generate` requires TTY
`bun run db:generate` prompts interactively when it detects camelCase ↔ snake_case column name ambiguity. This is a drizzle-kit limitation in non-TTY environments (CI, Claude Code). The schema is TypeScript-valid (confirmed by `bun run typecheck` passing). The migration must be generated in an interactive terminal session.

### Test design
- Schema tests use `Symbol.for("drizzle:Name")` and `Symbol.for("drizzle:Columns")` — drizzle-orm's internal runtime introspection symbols — to validate table names and column presence without a DB connection.
- The `createAuth` factory test validates only the function signature. Calling `createAuth({database: {}})` causes better-auth to asynchronously throw `BetterAuthError: Failed to initialize database adapter` as a background task after the test completes, which is expected behaviour with a mock adapter. The factory is validated at the type level by `bun run typecheck`.
