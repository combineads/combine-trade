## Patterns discovered

- **[New] API DI 패턴: RouteDeps 교차 타입** — 각 라우트 모듈이 자체 Deps 타입을 정의하고, RouteDeps = Health & SymbolStates & Positions & ... 교차 타입으로 합성. 새 라우트 추가 시 타입만 추가하면 됨.
- **[New] Bun 네이티브 대체** — Bun.password(bcrypt 대체) + hono/jwt(jsonwebtoken 대체)로 외부 의존성 2개 제거. Bun 에코시스템이 충분히 성숙.
- **[Recurring] DI 패턴의 테스트 이점** — API 서버도 daemon과 동일한 DI 패턴. mock deps 교체만으로 E2E 테스트 작성. 144개 API 테스트 모두 실제 Bun.serve 기동.
- **[Recurring] optional DI 하위 호환** — DaemonDeps.apiServer를 optional로 추가하여 기존 14개 daemon 테스트 변경 없이 통과.
