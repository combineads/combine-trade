## Patterns discovered

- **[New] 파라미터 교정은 API 변경 최소화가 핵심** — 기존 함수 시그니처를 유지하면서 내부 로직만 변경하면 파급 범위를 줄일 수 있음. T-10-002(등호), T-10-008(normalizer)이 좋은 예.
- **[Recurring] 타입 확장의 ripple effect** — AllIndicators에 필드 추가 시 모든 사용처 갱신 필요. 선택적 필드(bb4_1h?, prevSma20?)로 기본값 null 처리하면 기존 코드 영향 최소화.
- **[Recurring] DI 패턴의 테스트 이점** — 15개 모듈 교정에서 mock 교체만으로 기존 테스트 유지. EP-09에서 확립된 DI 패턴이 교정 작업에서도 효과적.
